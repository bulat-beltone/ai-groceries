# Мини-инструмент «Продукты» (ai-groceries) — спецификация

Документ описывает **текущее поведение** приложения (одна страница `index.html`, бэкенд опционально) и **ожидаемое поведение** по функциям списка, категорий, ИИ и перевода.

---

## 1. Назначение

- Ведение **списка покупок** на русском интерфейсе.
- **Автоматическая раскладка** позиций по отделам супермаркета с помощью ИИ (OpenAI через свой бэкенд — локальный Node или Cloudflare Worker).
- Настраиваемые **категории отделов**, ручная **коррекция отдела** после ИИ.
- Оформление названий при **вводе нескольких позиций через запятую** (или `;`).
- Опциональный **перевод названия** на выбранный язык (немецкий, английский, испанский), отображаемый под строкой.

Секреты API **никогда** не хранятся в HTML; на статическом хостинге используется Worker с секретом.

---

## 2. Техническая архитектура

| Компонент | Роль |
|-----------|------|
| `index.html` | UI, логика, `localStorage` (`gr7`). |
| `server.js` (порт по умолчанию 5173) | Отдаёт `/`, проксирует `POST /api/classify`, `POST /api/translate` к OpenAI при наличии `OPENAI_API_KEY`. |
| `worker/index.js` | Cloudflare Worker: те же маршруты + CORS для фронта на Pages. |
| `lib/handle-classify-request.js` | Разбор JSON, вызов классификации, валидация категорий. |
| `lib/handle-translate-request.js` | Разбор JSON, вызов перевода короткой строки. |
| `lib/openai-classify.js` / `lib/openai-translate.js` | Вызовы Chat Completions (`gpt-4o-mini`). |

Переменная во фронте: `CLASSIFY_API_BASE` — базовый URL Worker (без `/` в конце). Локально при `localhost` / `127.0.0.1` / `::1` запросы идут на **тот же origin** (`/api/...`), без обязательного Worker.

## 3. Данные в браузере (`localStorage`, ключ `gr7`)

Сохраняется JSON со следующими полями (имена могут отсутствовать у старых версий — тогда подставляются значения по умолчанию):

| Поле | Описание |
|------|----------|
| `items` | Массив позиций (см. ниже). |
| `hist` | История введённых названий для подсказок. |
| `uid` | Счётчик id новых позиций. |
| `cats` | Массив категорий `{ id, name, icon }`. |
| `catIdSeq` | Счётчик для генерации новых `id` вида `cat_N`. |
| `translateOn` | Включён ли показ переводов (`boolean`). |
| `translateLang` | Язык перевода: `de` \| `en` \| `es`. |

### 3.1. Позиция (`item`)

| Поле | Описание |
|------|----------|
| `id` | Числовой уникальный id. |
| `name` | Название продукта (после нормализации регистра — см. §5). |
| `checked` | Отмечено купленным. |
| `catId` | Id категории из `cats`. |
| `pending` | Идёт классификация ИИ; пока `true`, ответ ИИ может обновить отдел, **если пользователь не выбрал отдел вручную**. |
| `addedAt` / `checkedAt` | Метки времени для сортировки. |
| `tr` | Строка перевода (если включён перевод). |
| `trLang` | Язык, на который переведено (`de`/`en`/`es`). |
| `trPending` | Идёт запрос перевода. |
| `_trFetching` | Внутренний флаг, чтобы не дублировать параллельные запросы перевода для одной строки. |

## 4. Существующие функции (базовое поведение)

### 4.1. Ввод и список

- Поле «Что купить?», кнопка «Добавить», Enter добавляет.
- **Несколько позиций за раз**: строка режется по **запятой или точке с запятой**; каждая непустая часть становится отдельной позицией (после trim и **нормализации регистра** — §5).
- Подсказки из `hist` при вводе.
- Дубликат по имени (без учёта регистра): если позиция уже есть и вычеркнута — снова в активный список; если активна — повтор не добавляется.
- Чекбокс и клик по названию переключают «куплено»; удаление строки крестиком; «Очистить всё».

### 4.2. Классификация ИИ (`POST /api/classify`)

**Тело запроса (JSON):**

```json
{
  "name": "<строка позиции>",
  "categories": [
    { "id": "<латиница>", "name": "<отображаемое имя отдела>" }
  ]
}
```

**Ответ:** `{ "catId": "<один из id>" }` (при ошибке может быть `error`, подставляется запасной отдел).

На бэкенде ответ модели разбирается и приводится к одному из разрешённых `id` (`parse-cat-id.js`: границы слов, токены).

**Ограничения фронта:**

- `file://` или отсутствие URL API: ИИ не вызывается; позиция получает запасную категорию (через `catById('other')` при наличии такой категории).
- Если пользователь **уже выбрал отдел** вручную (`pending === false`), ответ ИИ **не перезаписывает** выбор.

### 4.3. Вкладка «Категории»

- Просмотр и редактирование списка отделов: эмодзи, название, стабильный **`id`** (латиница).
- Добавление новой категории (`cat_N`), удаление (не последнюю), при удалении товары переносятся на другой отдел (предпочтительно `other`, иначе первая подходящая).
- Категории по умолчанию задаются константой `DEFAULT_CATS`, если в хранилище ещё нет своего списка.

### 4.4. Сортировка и секции UI

- Секция «Определяю…» — позиции с `pending === true`.
- Далее блоки по категориям с иконкой и счётчиком.
- Пустой список — плейсхолдер с подсказкой.

---

## 5. Нормализация названия при добавлении

- После разбиения строки по `,` / `;` каждая часть **обрезается по краям**.
- Применяется правило **«первая буква заглавная, остальные строчные»** ко всей строке названия (`prettyProductName`), чтобы ввод вида `яйца, молоко, сахар` отображался как **`Яйца`**, **`Молоко`**, **`Сахар`**.
- Учитывается Unicode (кириллица и латиница через стандартные `toLowerCase` / `toUpperCase`).

---

## 6. Ручной выбор категории (новое требование)

- У **каждой** позиции в списке есть **выпадающий список отделов** (`select`).
- Пока ИИ не вернул отдел и не выбрано вручную (`pending && !catId`), первая опция — неактивная подпись вроде «Отдел…».
- При выборе конкретного отдела:
  - `catId` сохраняется;
  - `pending` сбрасывается в `false`;
  - ответ классификатора **не меняет** уже зафиксированный пользователем отдел (см. §4.2).

---

## 7. Перевод (новое требование)

### 7.1. UI

- Блок **«Перевод»**: чекбокс и выбор языка: **Немецкий** (`de`), **Английский** (`en`), **Испанский** (`es`).
- Пока чекбокс выключен, список языков может быть неактивен; подсказка, что нужен тот же бэкенд, что и для отделов.

### 7.2. Отображение

- Если перевод включён и для строки есть результат:
  - под основным названием показывается **вторая строка**: серый цвет, **курсив**;
  - при ожидании ответа — индикатор вроде «…».

### 7.3. API (`POST /api/translate`)

**Тело запроса:**

```json
{
  "text": "<строка названия>",
  "lang": "de" | "en" | "es"
}
```

**Ответ:** `{ "text": "<перевод>" }`; при ошибке может быть `error`, текст перевода пустой.

Модель: краткий промпт «перевести название товара для списка покупок на целевой язык», без лишнего текста в ответе.

### 7.4. Логика на клиенте

- Перевод запрашивается только при включённом чекбоксе и доступном `translateEndpoint()` (локально или Worker); для `file://` перевод не выполняется.
- При смене языка сбрасываются сохранённые переводы и запрашиваются заново для текущего языка.
- Для каждой позиции избегается дублирование параллельных запросов (`_trFetching`).

---

## 8. Контракты API (сводка)

| Метод и путь | Назначение |
|--------------|------------|
| `POST /api/classify` | Классификация отдела по названию и списку категорий. |
| `POST /api/translate` | Перевод короткой строки (название позиции) на выбранный язык. |
| `OPTIONS` на те же пути | CORS preflight для Worker. |

Заголовки ответов: `Content-Type: application/json; charset=utf-8`, CORS `*` для разрешённых методов.

---

## 9. Ограничения и ожидаемое развёртывание

- Для работы ИИ на проде (GitHub Pages и т.п.) нужен **задеплоенный Worker** с секретом `OPENAI_API_KEY` и актуальный `CLASSIFY_API_BASE` в `index.html`.
- После добавления маршрута `/api/translate` Worker нужно **переразвернуть** (`wrangler deploy`), чтобы перевод работал не только локально.
- Локально: `OPENAI_API_KEY` в `.env` / `.dev.vars` или переменная окружения (см. `SETUP.md` / `.env.example`).

---

## 10. Краткий чеклист поведения для тестирования

1. Добавить «Молоко, хлеб» — две строки, с **заглавной** первой буквой в каждой.
2. Дождаться или выбрать отдел; сменить отдел в `select` — отдел сохраняется, ИИ не перебивает после ручного выбора.
3. Вкладка «Категории» — добавить/переименовать/удалить отдел; новые позиции получают актуальный список для ИИ.
4. Включить «Перевод», выбрать язык — под названиями появляются серые курсивные строки после ответа API.
5. Сменить язык — переводы обновляются.
6. Локально без ключа — отделы/переводы деградируют предсказуемо (ошибка в консоли, запасные значения).
